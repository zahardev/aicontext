## Objective:
Create a minimal, tool-agnostic `/web-inspect` skill for AIContext that teaches AI assistants to use playwright-cli for browser-based investigation. Headed mode by default for security. Single concise prompt — not a copy of the upstream Microsoft skill.

## Context:

### Design Decisions:
- **Skill name: `web-inspect`** — self-descriptive, tool-agnostic, enables future pattern (`mobile-inspect`, etc.)
- **Bundle by default** — ~90% of users are web developers, no plugin/extension system needed
- **Our own skill** — minimal, focused on investigation workflow. Not a fork of upstream's 8-file skill
- **Single prompt file** — advanced commands discoverable via `playwright-cli --help [command]`
- **Headed by default** — user sees the browser, can login manually, no credential sharing with AI
- **Auth flexibility** — default: user logs in manually in headed browser. Optional: `state-save`/`state-load` for repeated sessions — user decides, AI doesn't ask for credentials
- **Awareness line in `standards.md`** — always-loaded one-liner so AI proactively suggests `/web-inspect` when it detects web UI work. The prompt itself only loads on invocation.
- **OS-aware install** — prompt includes system-specific install instructions
- **Snapshot-first workflow** — `snapshot` gets full page structure with refs, then target elements by ref (bulk approach, not multiple individual lookups)

### Best Practices to Include:
- **Snapshot before acting** — always `snapshot` first to get refs, don't guess selectors
- **One action, one snapshot** — re-snapshot after interactions that change the page
- **Console first for debugging** — `console error` before visual inspection to catch JS errors
- **Screenshots for visual issues** — when layout/styling is the question, screenshot > snapshot
- **Auth handling** — default headed (user logs in), optional `state-save`/`state-load` for speed

### Core Workflow:
1. `open --headed <url>` — always headed by default
2. `snapshot` — get page structure with element refs
3. `click`/`fill`/`select` by ref — interact with elements
4. `console` — check for JS errors
5. `screenshot` — capture current state
6. `eval` — run JS when needed
7. For everything else: `playwright-cli --help [command]`
