# Task 1.4.0: Skills, PR Workflow, and Framework Improvements

## Created:
March 1, 2026

## Objective:
Port battle-tested improvements from Translitera back into the AIContext framework: convert prompts to invocable Claude Code skills, add PR workflow tooling, improve data directory structure, upgrade agent model defaults, and deprecate redundant agents. Skills coexist with prompts to keep Cursor/Copilot support.

## Functional Requirements:
- Ship 8 Claude Code skills alongside existing prompts (coexist for Cursor/Copilot compatibility)
  - 5 converted from prompts: `/start`, `/task`, `/review`, `/check-plan`, `/after-step`
  - 3 new: `/next` (step completion flow), `/pr` (draft PR), `/pr-review-check` (triage PR reviews)
  - Skip `/dev-up` — project-specific, not framework material
- Ship PR workflow scripts in Node.js (fetch unresolved review threads, resolve threads) with graceful `gh` CLI detection
- Improve data directory with proper `.gitignore` and documented subdirectory conventions
- Deprecate `pr-review-summarizer` agent (replaced by `/pr-review-check` skill + scripts)
- Change agent model defaults from haiku to sonnet (reviewer to opus), with interactive haiku downgrade during init
- Update `init`/`update` to distribute skills, scripts, and handle deprecated agents

## Technical Considerations:
- Database changes needed? No
- API changes needed? No
- Breaking changes? No — all additive (deprecated agent removed during update)
- Third-party integrations affected? GitHub CLI (`gh`) required for PR scripts
- Performance implications? None
- Security considerations? PR scripts interact with GitHub API — requires `gh auth login`
- **PR scripts**: Rewritten in Node.js (zero extra deps — users already have Node from npm)
- **Cross-tool compatibility**: Skills are Claude Code only. Prompts remain for Cursor/Copilot. Scripts are tool-agnostic.
