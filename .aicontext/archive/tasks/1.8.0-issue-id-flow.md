# Task 1.8.0: Issue ID Flow in create-task

## Created:
April 12, 2026

## Spec:
[spec-1.8.0.md](../specs/spec-1.8.0.md)

## Objective:
When `task_naming.pattern` contains `{issue_id}`, offer a three-option menu instead of just asking for the number — so users can create an issue on the spot or skip it entirely.

## Deliverables:
- [x] `create-task.md` presents three options when `{issue_id}` is in the pattern: provide ID, create issue via `/draft-issue`, or skip (orphan task)
- [x] No regression for patterns without `{issue_id}`

## Plan:

### Step 1: Update create-task.md issue ID handling
- [x] Replace the current "ask for issue number" logic with a three-option menu: 1) Yes — here it is, 2) No — create an issue, 3) No — create orphan task without issue
- [x] Option 2 delegates to `draft-issue.md`, captures the issue number, and continues task creation
- [x] Option 3 creates the task with `{issue_id}` omitted or replaced with a placeholder per existing naming logic

## Completion Notes:

Prompt-only change to `create-task.md`. `start-feature.md` and `plan-tasks.md` have the same `{issue_id}` gap (pre-existing, not in scope).
