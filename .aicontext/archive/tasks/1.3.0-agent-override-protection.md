# Task 1.3.0: Agent Override Protection

## Created:
February 25, 2026

## Objective:
Prevent framework agent files from silently overwriting existing user agents during `init` and `update`. Add interactive per-file prompting and an explicit `--override-agents` flag.

## Functional Requirements:
- During `init`/`update`, check each framework agent against existing files at destination
- If agent file already exists: prompt user interactively "agents/X.md already exists. Override? (y/N)" (default No)
- If `--override-agents` flag is passed: override all without prompting
- If `--yes` flag is passed without `--override-agents`: skip existing agents (safe default, no interactive prompt)
- New framework agents (not yet at destination) are always copied without prompting
- Non-framework user agent files are never touched
- `.claude/CLAUDE.md` is always overwritten (framework entry point)
