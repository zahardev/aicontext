## Completion Notes:

Pivoted from systematic 40-file verbosity audit to manual per-prompt review because manual review catches bugs and cross-prompt consistency issues that a pure verbosity pass would miss. Scope was the prompts opened during the review session; remaining prompts can be revisited later if needed.

**Cross-cutting findings and fixes:**

- **Filename ambiguity** — three prompts had verb-verb / noun-noun ambiguity; renamed to verb-noun: `review-plan` → `review-task-plan`, `review-scope` → `detect-review-scope`, `update-check` → `check-update`. Old names added to `DEPRECATED_PROMPTS` / `DEPRECATED_SKILLS`.
- **Tool-specific assumptions generalized** — "IDE-opened file", "IDE selection", and `AskUserQuestion` were mentioned in prompts as if Claude Code was universal. Generalized to cover Claude Code, Cursor, Codex, Copilot across [identify-task.md](../prompts/identify-task.md), [add-step.md](../prompts/add-step.md), [detect-review-scope.md](../prompts/detect-review-scope.md), [generate.md](../prompts/generate.md), [create-task.md](../prompts/create-task.md), [add-idea.md](../prompts/add-idea.md).
- **Subagent cross-tool gaps** — [run-task.md](../prompts/run-task.md) and [step-loop.md](../prompts/step-loop.md) subagent references lacked the "(Claude Code only; run inline in other tools)" fallback. Added.
- **Commit logic regressions** — [run-step.md](../prompts/run-step.md) and [do-it.md](../prompts/do-it.md) had a Step 4 "Commit?" ask that ignored `commit.mode`, contradicting the documented policy in [run-task.md:26](../prompts/run-task.md#L26). Removed.
- **Dead config fields** — [release.template.md](../templates/release.template.md) had three fields never referenced (`version_source`, `base_branch`, `changelog_format`). Dropped.
- **Inline format duplication** — [review-criteria.md](../prompts/review-criteria.md) duplicated the code-review template format; removed the duplication and made [code-review.template.md](../templates/code-review.template.md) the single source of truth. Unified severity vocabulary (`critical / major / minor`).
- **Token savings** — web-inspect install section extracted to [install-playwright-cli.md](../prompts/install-playwright-cli.md), loaded only when the tool is missing (~80 tokens saved per `/web-inspect` invocation).
- **Info density rule for prompt topic lists** — added clause in `standards.md`: *"Prompt topic lists are candidates, not required sections — omit any with nothing to report"* (triggered by the empty "Spec drift" section I initially rendered in `/check-task` output).
- **New task surfaced** — unified step lifecycle config (`after_step` / `after_task` in `config.yml`, tri-state values, ask-upfront UX) captured as [1.7.0-step-lifecycle-config](1.7.0-step-lifecycle-config.md) with full spec section.
- **Brief template slimmed (Step 2)** — dropped `How to Use This File` (~10 lines of meta-instruction duplicated across every brief), `References` (8 lines, derivable from task path), `File References` (duplicated `git log --follow`), and dead `Bugs & Issues` + `Testing` placeholders. Kept the three sections with unique value: `Codebase Patterns`, `Gotchas`, `Decision Overrides`. Template went from 36 → 11 lines (~70% reduction). Each brief read in subsequent prompts saves the boilerplate cost. Existing briefs were not migrated — they're gitignored per-task scratch files, mostly on closed tasks.
