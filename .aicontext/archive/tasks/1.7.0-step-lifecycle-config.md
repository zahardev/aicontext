## Objective:
Unify step/task lifecycle configuration into `config.yml` under `after_step` and `after_task` sections. Replace the Quality Checks Timing Table in `process.md` and the fragmented `commit.mode` / `commit.finish_action` keys. Review and tests take **scope values** (`partial | full | false | ask`) â€” same vocabulary at both timings, no more `deep_review` / `full_tests` naming hack. Commit and push take boolean values (`true | false | ask`). `ask` fires upfront at `/run-step` or `/run-task` with a **two-stage prompt**: Stage 1 picks the action for this run (timing-specific recommendation first); Stage 2 asks `Save as default in config.yml? (y/N)`. Once answered, the run proceeds autonomously; no mid-flow interruptions for configured actions. The `reviewer` subagent now receives an explicit corpus (working-tree / last-commit / branch diff) computed from commit state, so review works correctly regardless of commit cadence.

## Completion Notes:

**Shape evolved during implementation.** Started with tri-state `true/false/ask` covering 7 keys including `after_task.deep_review` / `full_tests`. Mid-task review surfaced the asymmetry (separate key names for what is really one action at two timings) and the underlying review-corpus gap. Reworked to unified `partial/full/false/ask` scope vocabulary at both timings, same keys (`review`, `tests`) on both sides. Rework was in-scope per the task objective ("unify").

**Review corpus rule** is a real prompt-level addition, not just a rename. `step-loop.md` and `run-task.md` now compute the corpus explicitly from commit state â€” working-tree for uncommitted steps, `HEAD^..HEAD` for committed steps, `{base-branch}...HEAD` + working tree for task close. Closes a latent gap that was invisible until per-step commits became configurable.

**Two-stage ask UX** (Stage 1: decision with timing-specific recommendation; Stage 2: save-as-default y/N) lets users save *any* answer, not just the recommended one. Stage 2 always fires (simpler than conditional firing, marginal extra keystroke).
