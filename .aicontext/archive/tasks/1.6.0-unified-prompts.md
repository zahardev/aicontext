- Moved all skill/agent instructions to `.aicontext/prompts/` as single source of truth

- Simplified commit logic: `/run-steps` only commits per-step, `/finish-task` is the single decision point for per-task commits via `finish_action`
- `/finish-task` now warns when `finish_action: nothing` but uncommitted changes exist
- Added `commit_body` option to Commit Rules (default: `true` — body with what/why + Co-Authored-By trailer)
- Strong enforcement: `commit_body: false` = subject only, no body/trailers/Co-Authored-By
- Co-author template: `Co-Authored-By: {ai} via AIContext` (agent replaces `{ai}` with model name)
- Added Commit Style section to `standards.md` explaining `commit_body` behavior
- Added commit rule descriptions to `project.template.md` and `project.md`
- Removed unsupported `commit+push+pr` finish_action option
- Updated `/prepare-release` to read specs → tasks → briefs → git diff (context-first approach)
- Renamed `pr-review-check` → `gh-review-check` (consistency with `gh-review-fix-loop`)
- Renamed `check-plan` → `review-plan` (better reflects review/improve behavior vs status check)
- Added `/challenge` skill — pause and push back before proceeding (questions, assumptions, concerns, better ideas)
- Commit body rules updated: body should explain *why*, not *what* — the diff shows what
