- [x] Replace §3 "Depth-cap for surprises" bullet with a single-sentence rule that asks whether the surprise belongs in an existing dimension (yes → allow 1–2 follow-ups; no → add to dimension map and continue breadth-first)
- [x] Ensure the rule cross-references §2 (live dimension map) implicitly via the decision test

### Step 3: Standalone purpose check with smart skip
- [x] Add a purpose-check sub-step at the top of §1 Before Asking: ask *"What are we trying to decide here?"* when the recent conversation has no clear topic
- [x] Smart-skip rule: if the most recent user turn(s) already mention a specific topic or the caller explicitly seeded one (e.g., `/start-feature` invocation), skip the purpose check
- [x] Do not ask when invoked via an orchestrator that provides the topic

- [x] Add one bullet under §3 Interview Rules: *"**5 Whys for diagnostics** — when drilling into a bug, failure mode, or root cause, the 5 Whys pattern (chained 'why' questions) is often the right depth-cap drilling sequence."*
