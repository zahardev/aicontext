# Task 1.7.0: Bulk overwrite question in aicontext update

## Created:
April 6, 2026

## Spec:
[spec-1.7.0-framework-improvements.md](../specs/spec-1.7.0-framework-improvements.md)

## Objective:
Replace the separate per-file "Override?" prompts for agents and skills (and the existing prompts overwrite question) with a single upfront "Override all existing framework files?" question during `aicontext update`. If yes, bulk overwrite everything — prompts, agents, skills, and codex skills — with no further per-file prompts. If no, fall through to the current per-file behavior.

## Plan:

### Step 1: Add bulk question to `update()`
- [x] After showing the "The following will be updated" list and before the "Continue?" confirm, add: `"Override all existing framework files (prompts, agents, skills)? (Y/n): "`
- [x] Only ask when there are existing files that would conflict — skip if nothing would be overwritten
- [x] If Y: set `overwritePrompts = true`, skip the existing per-prompts question, pass `overrideAgents = true` and `overrideSkills = true` to all `copyFramework*` calls
- [x] If N: fall through to existing flow (existing prompts question + per-file agents/skills prompts)
- [x] Skip the bulk question entirely if `--override-agents` or `--override-skills` CLI flags are already set — existing flags take precedence

### Step 2: Update tests
- [x] Add test: bulk Y → all files overwritten, no per-file prompts fired
- [x] Add test: bulk N → falls through to existing per-file behavior (covered by hasExistingFrameworkFiles unit tests + existing per-file tests)
- [x] Add test: no existing files → bulk question not asked

## Completion Notes:

- Added `hasExistingFrameworkFiles()` helper — checks agents, skills, codex skills, and prompts; bulk question only fires when files exist
- Bulk Y sets `overrideAgents`, `overrideSkills`, `overwritePrompts` and skips the per-prompts question and the "Continue?" confirm
- Bulk N falls through to existing per-file behavior unchanged
- Question skipped when `--override-agents`/`--override-skills` CLI flags are pre-set
- Manual testing caught two UX issues: (1) "Continue?" appeared redundantly after bulk Y — fixed by skipping it; (2) question wording didn't explain what N does — reworded to "Y = update everything at once, N = choose file by file"
- File list now dynamically reflects override state for agents/skills lines
