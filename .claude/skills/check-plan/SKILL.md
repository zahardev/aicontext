name: check-plan
description: Validate the current implementation plan for dependency issues, gaps, and over-engineering
disable-model-invocation: true
---

Review the current plan for:

1. **Dependency issues** - Does any step depend on a later step?
2. **Missing steps** - Any gaps in the implementation flow?
3. **Over-engineering** - Unnecessary complexity or abstractions?

If no issues: confirm "Plan is valid" and summarize the implementation order.
